## ワイヤレスLAN

### 規格と周波数帯の使い分け

WLANの標準規格は **IEEE 802.11** です。  
特に周波数帯の特性の違いを理解しておくことが重要です。

| 周波数帯 | メリット | デメリット/特徴 | CCNAでの要点 |
| :--- | :--- | :--- | :--- |
| **2.4 GHz** | **遠くまで届く**（障害物に強い） | **遅い、混雑しやすい**（電子レンジなどと干渉） | 非重複チャネルは**3つ**（1, 6, 11）。干渉を避ける設計が重要。 |
| **5 GHz** | **高速で安定している** | 遠くまで届きにくい（障害物に弱い） | **チャネル数が多く**、干渉が少ないため、大容量通信に適する。 |

-----

### 動作原理：CSMA/CA (衝突回避)

有線LAN（イーサネット）との決定的な違いは、データの送信制御方式です。

  * **有線LAN（イーサネット）:** **CSMA/CD** (衝突**検知**)
      * 衝突を検知したら、送信を停止し、再送を試みる。
  * **ワイヤレスLAN (WLAN):** **CSMA/CA** (衝突**回避**)
      * 無線環境では衝突検知が難しいため、送信前にチャネルが空いているかを確認し、**衝突が発生しないように試みます**。
      * 送信が成功したかどうかは、受信側からの応答（ACK）によって確認されます。  
      この仕組みが、有線LANに比べると効率が下がる一因です。

### APの管理モデル

アクセスポイント（AP）の管理方法には、主に2つのモデルがあります。

| モデル | 管理方法 | 利点 |
| :--- | :--- | :--- |
| **自律型AP**<br />(Autonomous AP) | AP単体で動作・設定を保持 | 小規模ネットワーク向け。APごとに個別の設定が必要。 |
| **集中管理型**<br />(Controller-Based) | **WLC** (Wireless LAN Controller) で一元管理 | 大規模ネットワーク向け。APの設定、ファームウェア更新、セキュリティポリシーなどをコントローラで一括管理でき、運用負荷が大幅に軽減される。 |

## ワイヤレスLANセキュリティ対策

WLANのセキュリティは、主に以下の2つの要素で成り立っています。

1.  **認証 (Authentication):** 接続してくるユーザーやデバイスが「正当な利用者であるか」を確認するプロセス。
2.  **暗号化 (Encryption):** 通信内容が第三者に傍受されても、内容が読み取れないように変換するプロセス。

-----

### 暗号化プロトコル：WPA3への移行

不正アクセスを防ぐために、通信データを暗号化します。  
プロトコルは時代とともに進化しており、古いものは危険です。

| プロトコル | 世代 | セキュリティレベル | 特徴 |
| :--- | :--- | :--- | :--- |
| **WEP / WPA** | 旧世代 | **非常に危険** | 現在は使用禁止。簡単に解読される。 |
| **WPA2** | 現行主流 | 高い | 企業や家庭で広く使われている。**AES**暗号化が基本。 |
| **WPA3** | 最新規格 | **最高** | WPA2の脆弱性を改善。特にパスワード推測攻撃（オフライン辞書攻撃）への対策が強化されている。 |

-----

### 認証方式の使い分け（PSK vs 802.1X）

「誰をネットワークに入れるか」を決める認証には、主に2つの方式があります。

| 認証方式 | WPA2/WPA3-**PSK**<br />(Pre-Shared Key) | WPA2/WPA3-**Enterprise**<br />(802.1X/EAP) |
| :--- | :--- | :--- |
| **適用規模** | 家庭、小規模オフィス | **大規模な企業・大学** |
| **認証方法** | **共通パスワード**（すべてのユーザーが同じキーを使用） | **ユーザーIDとパスワード**（ユーザーごとに個別の認証情報を使用） |
| **セキュリティ** | 漏洩リスクあり | **非常に強固** |
| **必要な設備** | アクセスポイントのみ | **RADIUSサーバー**が必要 |

### 802.1X/EAPの仕組み

企業向けの**802.1X認証**では、接続要求があった際にアクセスポイントがゲートウェイの役割を果たし、**RADIUS** (Remote Authentication Dial-In User Service)という専用の認証サーバーにユーザー情報を問い合わせて接続の可否を決定します。

-----

### その他の対策と管理

| 対策 | 目的 | 要点 |
| :--- | :--- | :--- |
| **不正AP対策** | 悪意のあるAPや設定ミスのAPを防ぐ | 攻撃者が設置した**不正アクセスポイント**（Rogue AP）を検知・遮断することはセキュリティ管理上、極めて重要。 |
| **MACフィルタリング** | 許可された端末のみ接続させる | 物理アドレス（MACアドレス）に基づいてアクセスを制御するが、MACアドレスは偽装が容易なため、**単独でのセキュリティ効果は低い**。 |
| **SSIDステルス** | SSIDの隠蔽 | SSIDをブロードキャストしないようにするが、ツールの使用でSSIDは容易に特定できるため、**セキュリティ対策としての効果は限定的**。 |
