# 2. Cisco IOS コマンド

<details>
    <summary>概要</summary>
    <div>

## Cisco IOS

Cisco Systems社のネットワーク機器（ルーターやスイッチ）を動かすためのOSです。  
WindowsやMacのようにマウスでクリックする画面（GUI）ではなく、**Cisco CLI**を使って操作します。

-----

### 最も重要な3つのモード

Cisco機器の操作で初心者が一番混乱するのが**今、自分がどのモードにいるか**です。  
プロンプト（行の先頭の記号）を見ることで、自分の立ち位置を把握します。

| モード名 | プロンプト | 権限レベル | できること |
| :--- | :--- | :--- | :--- |
| **ユーザーEXECモード** | `Router>` | 低 | 機器の状態を少しだけ見れる。設定変更は不可。 |
| **特権EXECモード** | `Router#` | 中 | 全ての情報を見れる。設定モードへの入り口。 |
| **グローバルコンフィグモード** | `Router(config)#` | 高 | 機器全体の設定（名前変更など）を変更できる。 |

:::tip ポイント
設定を変えたい時は、必ず `>` → `#` → `(config)#` と階段を登っていく必要があります。
:::

-----

### 必須コマンド7選

初心者が最初に覚えるべきコマンドを、用途別に整理しました。
カッコ内の**略称**を使ってもコマンドは通ります（現場では略称を使うのが一般的です）。

| カテゴリ | コマンド （略称） | 動作・意味 | 使用場面 |
| :--- | :--- | :--- | :--- |
| **移動** | **`enable`**(`en`) | **特権モードへ入る**（`>` → `#`） | ログイン直後、最初に必ず入力します。 |
| **移動** | **`configure terminal`**(`conf t`) | **設定モードへ入る**（`#` → `(config)#`） | 名前変更やIP設定など、設定を変更したい時に入力します。 |
| **移動** | **`exit`** | **一つ前のモードに戻る** | 作業が終わって、段階的に戻りたい時。 |
| **移動** | **`end`** | **特権モードへ一気に戻る** | 深い階層から一発で `Router#` に戻りたい時。 |
| **確認** | **`show running-config`**(`sh run`) | **現在の設定をすべて表示** | 設定が反映されているか確認する時。※最も頻繁に使います。 |
| **確認** | **`show ip interface brief`**(`sh ip int br`) | **ポートの状態を一覧表示** | ケーブルが正しく認識されているか、IPアドレスが入っているか確認する時。 |
| **疎通** | **`ping [相手のIPアドレス]`** | **通信確認を行う** | 相手と繋がっているか確認する時。 |

-----

### 補助機能

コマンドを丸暗記していなくても、この2つのキー操作を知っていれば自力で答えにたどり着けます。

| 機能・キー | 役割 | 具体的な使い方 |
| :--- | :--- | :--- |
| **?** | **候補の表示**：次に何を打てるか教えてくれる | **`show ?`** と入力→ `show` の後に続けられるコマンドの一覧がズラッと表示されます。 |
| **Tab** | **自動補完**：途中まで打った文字を完成させる | **`conf`** と打ってTabキーを押す→ **`configure`** と自動的に文字が補完されます。|

-----

### 実際の操作フロー（演習イメージ）

以下は、実際にルーターにログインして設定を確認するまでの流れです。

```text
Router> enable                  <-- ① 特権モードに入る
Router# configure terminal      <-- ② 設定モードに入る
Router(config)# hostname R1     <-- ③ 名前を「R1」に変更
R1(config)# exit                <-- ④ いったん戻る
R1# show ip interface brief     <-- ⑤ 接続状態を確認
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0     192.168.1.1     YES manual up                    up      
R1#
```
