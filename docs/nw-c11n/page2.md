## LAN（Local Area Network）
家やオフィスの中など、限られた範囲のネットワーク。  
インターネットという「外の世界」に出る前の、自分たちの「内側の世界」のことです。

### 主な構成要素
- 有線LAN（イーサネット）： ケーブル（LANケーブル）を物理的に挿して繋ぐ方式。安定して速い。
- 無線LAN（Wi-Fi）： 電波を使って繋ぐ方式。ケーブルがなく便利。
- ハブ / スイッチ： パソコンやプリンターなど、複数の機器を束ねる「コンセントのタコ足配線」のような役割を持つ中継器。

:::tip ポイント
LANの中にいる機器同士（例：自分のPCと隣の席の人のPC、またはオフィスのプリンター）は、外のインターネットを通らずに直接やり取りができます。
:::

## OSI参照モデルとTCP/IPモデルの比較と背景

### 歴史的背景：なぜ2つのモデルが存在するのか

| モデル | 考案された主体と背景 | 目的と特徴 |
| :--- | :--- | :--- |
| **OSI参照モデル** | **ISO**（国際標準化機構）が1980年代に策定。当時、ベンダー（メーカー）ごとに独自の通信方式を採用しており、異なる機器間の通信が困難だったため。 | **理想の設計図**（理論的標準）。ネットワーク機能の動作を理解するための、厳密で詳細な**共通言語**として開発された。 |
| **TCP/IPモデル** | **DARPA（米国国防総省高等研究計画局）**がARPANET（インターネットの前身）のために開発。理論より**実用性**と**堅牢性**が最優先された。 | **インターネットの事実上の標準**（デファクトスタンダード）。非常にシンプルで柔軟なため、インターネットの爆発的な普及を支えた。 |

### 階層の比較と対応関係

TCP/IPは実用を優先したため、OSI参照モデルのいくつかの層を統合しています。

| OSI参照モデル (7層) | TCP/IPモデル (4/5層) | 役割とプロトコル例 |
| :--- | :--- | :--- |
| **第7層：アプリケーション層** | **アプリケーション層** | アプリケーションへのサービス提供。HTTP, SMTP, DNS, FTP |
| **第6層：プレゼンテーション層** | *（統合）* | データの表現形式、暗号化/復号 (SSL/TLSなどが機能) |
| **第5層：セッション層** | *（統合）* | 通信セッションの確立、維持、切断 |
| **第4層：トランスポート層** | **トランスポート層** | 信頼性、再送制御、ポート番号管理。**TCP**, **UDP** |
| **第3層：ネットワーク層** | **インターネット層** | IPアドレッシング、経路選択（ルーティング）。**IP**, ICMP |
| **第2層：データリンク層** | **ネットワークアクセス層** | 物理アドレス（MACアドレス）、フレーム制御。Ethernet, Wi-Fi |
| **第1層：物理層** | **ネットワークアクセス層** | 物理的な信号、電気的特性、ケーブル。ケーブル、ハブ |



### なぜ両方を知る必要があるか

* **TCP/IP**は、現在利用しているインターネットの「**仕組み**」そのものです。
* **OSI参照モデル**は、障害発生時や新しい技術を学習する際に、どの機能がどのレベルで問題を起こしているかを**整理・特定**するための「**共通の思考フレームワーク**」として現在も不可欠です。

## TCP/IP
世界標準の通信ルール（プロトコル）のセット。  
メーカーやOS（WindowsやMac）が違っても通信ができるのは、全員がこの「TCP/IP」という共通のルールを守っているからです。

### 2つの要素
実際は多くのプロトコルの集まりですが、代表的な2つが名前の由来です。

#### IP（Internet Protocol）
- 役割： 「宛先（住所）まで届ける」
- キーワード： IPアドレス（インターネット上の住所）
- イメージ： 荷物に「宛名ラベル」を貼って、正しい家に届ける配送ルートを決める役割。

#### TCP（Transmission Control Protocol）
- 役割： 「正確に届いたか確認する」
- キーワード： 信頼性、確認応答
- イメージ： 荷物が壊れていないか確認し、届いたら受取サインをもらう役割。  
もし途中で荷物が消えたら、再送を依頼します。
