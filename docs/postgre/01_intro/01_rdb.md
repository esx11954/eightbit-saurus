# 1-1. データベースの種類とリレーショナルデータベース

## データベースとは

データベース（DB）は、大量のデータを効率よく保存し、高速に取り出すための仕組みです。  
単なるファイル保存との違いは、**検索・更新・削除を高速かつ安全に行える**点にあります。

## データベースの種類

データベースにはいくつかの種類（モデル）があります。

| 種類 | 概要 | 代表製品 |
| :--- | :--- | :--- |
| **リレーショナル型 (RDB)** | 表形式でデータを管理し、SQL で操作する。最も広く使われている | PostgreSQL, MySQL, Oracle |
| **ドキュメント型** | JSON などの形式でデータを保存する。スキーマが柔軟 | MongoDB, Firestore |
| **キーバリュー型** | キーと値のペアでデータを保存する。高速なキャッシュ用途に向く | Redis, DynamoDB |
| **グラフ型** | ノードとエッジでデータの関係を表現する | Neo4j |

本研修では **PostgreSQL**（リレーショナル型） を使用します。

## ExcelとRDBの違い

Excelは「表計算」ソフトであり、個別のデータを自由に配置できます。  
一方、RDBは**厳密なルール**に基づいてデータを管理します。

| 特徴 | Excel | RDB (PostgreSQL) |
| :--- | :--- | :--- |
| **データ形式** | 自由（1つのセルに数値と文字を混ぜられる） | 厳密（列ごとにデータ型が決まっている） |
| **同時操作** | 1人ずつ編集するのが基本 | 数百、数千人が同時にアクセス可能 |
| **データ量** | 数万行を超えると動作が重くなる | 数億行のデータも高速に処理できる |
| **関係性** | リンク切れが起きやすい | 「外部キー」により整合性が保証される |

## ACID特性（RDBの信頼性）

RDBは、データの正しさを守るために以下の4つの特性（ACID）を保証します。

| 特性 | 名称 | 内容 |
| :--- | :--- | :--- |
| **A** | Atomicity（原子性） | 処理が「全部完了」か「一切やらない」のどちらかになる |
| **C** | Consistency（一貫性） | DBのルール（型や制約）に違反するデータは保存されない |
| **I** | Isolation（独立性） | 同時に行われる他の処理に影響を与えない |
| **D** | Durability（永続性） | 一度保存されたデータは、システムが故障しても失われない |

:::info 集合論の考え方
RDBは「集合（セット）」の考え方をベースにしています。    
「特定の条件に一致するデータをひとまとめにする」という操作を、SQLという言語で行います。
:::

## PostgreSQLとは

PostgreSQL は、以下の特徴を持つオープンソースのRDBMSです。

- **信頼性**: ACID特性を完全に実装しており、金融系・基幹系システムにも採用される
- **拡張性**: カスタムデータ型・関数・インデックスを追加できる
- **標準準拠**: SQL標準への準拠度が高く、移植性に優れる
- **ライセンス**: PostgreSQLライセンス（商用利用も無料）
