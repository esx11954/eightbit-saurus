# 1-2. データベースにおける考慮事項とSQL

## データベース設計における考慮事項

データベースを設計・運用するうえで、以下の観点を意識する必要があります。

### 1. データ整合性

データに矛盾が生じないよう、ルール（制約）を設けます。

- **型制約**: 「年齢」列に文字列が入らないようにする
- **NOT NULL制約**: 必須項目を空欄にできないようにする
- **外部キー制約**: 存在しない部署IDを社員テーブルに登録できないようにする

### 2. パフォーマンス

大量データを扱うとき、検索速度が問題になります。

- **インデックス**: よく検索する列にインデックスを貼ることで検索を高速化する
- **正規化**: テーブルを適切に分割し、データの重複を排除する
- **クエリの最適化**: `EXPLAIN` コマンドで実行計画を確認し、遅いクエリを改善する

### 3. セキュリティ

データへのアクセスを適切に制御します。

- **ロール管理**: ユーザーごとに「読み取りのみ」「書き込み可」などの権限を付与する
- **スキーマ分離**: 公開データと機密データを別スキーマに分ける
- **接続制御**: `pg_hba.conf` で接続元IPやユーザーを制限する

### 4. 可用性・バックアップ

障害時のデータ保護を考慮します。

- **定期バックアップ**: `pg_dump` コマンドでスナップショットを取得する
- **WAL（Write-Ahead Logging）**: 障害時でもデータを復旧できるログ機構
- **レプリケーション**: 複数サーバーにデータを複製し、単一障害点をなくす

## SQLとは

SQL（Structured Query Language）は、リレーショナルデータベースを操作するための言語です。
プログラミング言語と異なり、**「何が欲しいか」を宣言的に記述する**のが特徴です。

```sql
-- 「何をする」かを宣言する（「どうやって取るか」は書かない）
SELECT emp_name FROM employees WHERE dept_id = 1;
```

## SQLの分類

SQLは目的によって4種類に分類されます。

| 分類 | 正式名称 | 主なコマンド | 役割 |
| :--- | :--- | :--- | :--- |
| **DDL** | Data Definition Language | `CREATE` / `ALTER` / `DROP` | テーブルなどの**構造**を定義する |
| **DML** | Data Manipulation Language | `SELECT` / `INSERT` / `UPDATE` / `DELETE` | テーブルの**データ**を操作する |
| **TCL** | Transaction Control Language | `COMMIT` / `ROLLBACK` / `BEGIN` | **トランザクション**を制御する |
| **DCL** | Data Control Language | `GRANT` / `REVOKE` | **アクセス権限**を管理する |

:::tip 本研修の学習順序
**DDL → DML → TCL → DCL** の順で学習します。
まず「箱（テーブル）を作る」DDLを学び、次に「中身を操作する」DMLへ進みます。
:::

## SQLの基本的な書き方

SQLのキーワードは大文字・小文字を区別しませんが、**キーワードを大文字**で書くのが一般的な慣習です。

```sql
-- 基本的なSELECT文の構造
SELECT  列名1, 列名2       -- 取得する列を指定
FROM    テーブル名          -- 対象テーブルを指定
WHERE   条件式             -- 絞り込み条件（省略可）
ORDER BY 列名 ASC;         -- 並び順（省略可）
```

```sql
-- 複数行にまたがるSQLはセミコロン(;)で終わる
SELECT emp_name, dept_id
FROM   employees
WHERE  dept_id = 2
ORDER BY emp_name ASC;
```

:::note コメントの書き方
```sql
-- 1行コメント

/*
  複数行
  コメント
*/
```
:::
