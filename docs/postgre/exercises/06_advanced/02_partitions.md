# 高度な操作演習2: パーティショニング

対応章: [7章 高度な操作](../../07_advanced/01_overview.md)

---

## 問題1 RANGEパーティショニング

アクセスログテーブルを月単位でパーティショニングしてください。

1. 以下の仕様で親テーブル `access_logs` を作成する
   - `log_id BIGSERIAL`
   - `user_id INTEGER`
   - `accessed_at TIMESTAMPTZ NOT NULL`（パーティションキー）
   - `path TEXT`

2. 2024年1月・2月・3月の子テーブルを作成する

3. 以下のデータを挿入し、それぞれ正しい子テーブルに振り分けられることを確認する

   ```sql
   INSERT INTO access_logs (user_id, accessed_at, path) VALUES
       (1, '2024-01-15 10:00:00+09', '/home'),
       (2, '2024-02-20 15:30:00+09', '/login'),
       (3, '2024-03-05 09:00:00+09', '/logout');
   ```

4. 親テーブルに対して `accessed_at` で絞り込むクエリを `EXPLAIN` で確認し、パーティションプルーニングが効いていることを確認する

## 問題2 LISTパーティショニング

地域ごとに注文データを管理するテーブルを作成してください。

1. `region TEXT NOT NULL` をパーティションキーとした `orders` テーブルを作成する
2. `east`（東京・千葉・神奈川）・`west`（大阪・京都・兵庫）・`other`（それ以外）の3つのパーティションを作成する
3. 各地域にデータを挿入して動作を確認する
4. 「未知の地域」（例: `'北海道'`）のデータは `other` に入ることを確認する

---

## 問題3 マテリアライズドビュー

部署別サマリーのマテリアライズドビューを作成し、リフレッシュを体験してください。

1. 以下の内容のマテリアライズドビュー `dept_summary` を作成する
   - 部署名・社員数・平均給与・最高給与

2. 社員の給与を変更する（例: 田中 太郎の給与を999999に変更）

3. ビューを参照して、変更が**反映されていないこと**を確認する

4. `REFRESH MATERIALIZED VIEW` を実行して、変更が反映されることを確認する

---

## 問題4 パーティション管理

問題1で作成した `access_logs` テーブルに対して、以下を実施してください。

1. 2024年4月の子テーブルを追加する
2. 2024年1月のデータを集計する（件数・ユーザー数）
3. 2024年1月のパーティションを `DROP TABLE` で削除し、親テーブルへのSELECTでも1月のデータが消えていることを確認する
4. 通常の `DELETE FROM access_logs WHERE accessed_at < '2024-02-01'` と比べて、何が速いのか、なぜ速いのかを説明する
