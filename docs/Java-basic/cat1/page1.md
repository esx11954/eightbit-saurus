# Javaの歴史・進化

## 1. はじめに  

Javaは、Sun Microsystems社（現Oracle社）が開発したオブジェクト指向プログラミング言語です。  
その特徴は、「**Write Once, Run Anywhere**（一度書けば、どこでも動く）」というコンセプトに集約されます。  
これは**Java仮想マシン**（JVM）という抽象化レイヤーを介することで、特定のプラットフォームに依存せず、多様な環境で同じプログラムを実行できることを意味します。  

## 2. Java誕生の背景と初期の歴史  

Javaは、当初「Oak」というプロジェクト名で、家電製品向けの組み込みシステム開発を目指して1991年にジェームズ・ゴスリン氏によって開始されました。  
しかし、1990年代半ばのインターネットの急速な普及とWebの登場により、その潜在能力が見出され、1995年に「Java」と改名されてWebアプレット技術として発表されました。  

| 年代 | イベント | 主な特徴・出来事 |
| :--- | :--- | :--- |
| **1991年頃** | Oakプロジェクト開始 | ジェームズ・ゴスリン氏が家電製品向けに開発を開始しました。  |
| **1995年** | 「Java」として発表 | インターネット時代のWebアプレット技術として注目を集めました。  |
| **1996年1月** | JDK 1.0リリース | 初期バージョンのJava開発キットです。  AWT（Abstract Window Toolkit）やAppletのサポートが中心でした。  |
| **1997年2月** | JDK 1.1リリース | JavaBeans、JDBC（データベース接続）、RMI（リモートメソッド呼び出し）などの重要なAPIが追加されました。  JIT（Just-In-Time）コンパイラの導入により実行性能が向上しました。  |

## 3. Java エンタープライズ

Javaの成熟期は、1998年末にリリースされた「Java 2」（内部バージョンはJDK 1.2）から本格化します。  
このバージョンから、Javaは単なる言語を超え、用途に応じたプラットフォーム群として整理されました。  

*   **J2SE (Java 2 Platform, Standard Edition):**  デスクトップアプリケーションやクライアントサイドの一般的なJava開発向けです。  
*   **J2EE (Java 2 Platform, Enterprise Edition):**  大規模なサーバーサイドアプリケーション（Webアプリケーション、エンタープライズシステムなど）開発向けです。  Servlet、JSP（JavaServer Pages）、EJB（Enterprise JavaBeans）などの技術が含まれます。  
*   **J2ME (Java 2 Platform, Micro Edition):**  組み込み機器やモバイルデバイス向けのJava開発向けです。  

| バージョン | リリース時期 | 主な特徴・目的 |
| :--- | :--- | :--- |
| **Java 2 (JDK 1.2)** | 1998年12月 | 「Java 2」という名称で、プラットフォームとして整理されました。  Swing（新しいGUIツールキット）やCollections Frameworkが導入されました。  HotSpot JVMが登場し、実行性能が飛躍的に向上しました。  |
| **J2EE** | 1999年 | サーバーサイドの標準化を進め、エンタープライズシステムの基盤となりました。  |
| **J2ME** | 1999年 | 組み込みやモバイルデバイス向けに、Javaの適用範囲を広げました。  |

## 4. Javaの成熟と大規模な進化  

Java 5（Tiger）以降、Javaは言語仕様とプラットフォームの両面で大きな進化を遂げ、現代のJavaプログラミングの基礎が確立されました。  
特にJava 8（Lambda）は、その後のJava開発のスタイルを大きく変える画期的なリリースとなりました。  

| バージョン | リリース時期 | 主な機能拡張 | 影響・意義 |
| :--- | :--- | :--- | :--- |
| **Java 5 (Tiger)** | 2004年9月 | **ジェネリクス (Generics)**、**アノテーション (Annotations)**、オートボクシング/アンボクシング、拡張for文、列挙型、可変長引数など | 現代Javaのプログラミングスタイルの基礎を築きました。  型安全性の向上とコードの簡潔化に大きく貢献しました。  |
| **Java 6 (Mustang)** | 2006年12月 | スクリプト言語対応 (JavaScript等)、JDBC 4.0、Webサービス関連APIの強化など | 安定性とパフォーマンスの向上に重点が置かれました。  |
| **Java 7 (Dolphin)** | 2011年7月 | try-with-resources文、ダイヤモンド演算子 (`<>`)、NIO.2（ファイルI/Oの強化）など | 細かい文法的な改善と、ライブラリの強化が行われました。  |
| **Java 8 (Lambda)** | 2014年3月 | **ラムダ式 (Lambda Expressions)**、**Stream API**、Defaultメソッド (インターフェース内実装)、新しい日付/時刻API (`java.time`)、Nashorn JavaScriptエンジンなど | 関数型プログラミングの概念が導入され、Javaプログラミングのパラダイムが大きく変わりました。  コードの記述量が減り、より宣言的なスタイルが可能になりました。  |

## 5. Oracleによる買収とリリースサイクルの変更  

2010年、Oracle社はSun Microsystems社を買収し、Javaの管理・開発の主体がOracle社へ移行しました。  
これにより、Javaの将来的な方向性やリリースモデルに大きな変更が加えられました。  

*   **リリースサイクルの変更:**  Java 9以降、これまでの数年ごとのメジャーリリースから、半年ごとのリリースサイクルへと移行しました。  
これにより、新機能がより迅速に提供されるようになりました。  
*   **LTS (Long Term Support) 版の導入:**  半年ごとのリリースとは別に、長期サポートが提供されるLTS版が指定されるようになりました。  エンタープライズ環境では、このLTS版が推奨されます。  
これまでのLTS版はJava 8、11、そして現在の17です。  

## 6. Java 9以降の主な進化（LTS版中心）  

新しいリリースサイクルになってからも、Javaは様々な新機能を取り入れ、進化を続けています。  
Java 17は現在のLTS版であり、多くのモダンな機能が導入されています。  

| バージョン | リリース時期 | 主な機能拡張 | 影響・意義 |
| :--- | :--- | :--- | :--- |
| **Java 9** | 2017年9月 | **Java Platform Module System (JPMS / Jigsaw)**、JShell (REPL環境)、Multi-Release JAR Files、HTTP/2 Client (incubator) など | Javaのプラットフォーム自体がモジュール化され、より小さく、より安全で、より保守性の高いアプリケーション構築が可能になりました。  |
| **Java 10** | 2018年3月 | ローカル変数型推論 (`var`) | コードの記述量を減らし、可読性を向上させます。  |
| **Java 11 (LTS)** | 2018年9月 | **HTTP Client API (標準化)**、ZGC (実験的GC)、Flight Recorder (標準化)、Lambdaパラメータの`var`など | 新しい標準HTTPクライアントが導入されました。  これにより、RESTfulなWebサービスとの連携が容易になりました。  |
| **Java 12-16 (非LTS)** | 2019年3月-2021年3月 | Switch Expressions (プレビュー→標準化)、Text Blocks (プレビュー→標準化)、Records (プレビュー→標準化)、Pattern Matching for `instanceof` (プレビュー→標準化) など | 次期LTS版に向けた、言語機能の実験と標準化が段階的に進められました。  |
| **Java 17 (LTS)** | 2021年9月 | **Sealed Classes (標準化)**、**Pattern Matching for `switch` (プレビュー)**、Foreign Function & Memory API (incubator)、Vector API (incubator) など | クラスの継承を厳密に制御するSealed Classesや、`switch`文でのパターンマッチングが導入され、より洗練されたコード記述が可能になりました。  JVMのパフォーマンスやネイティブ連携も強化されています。  |

## 7. まとめ  

Javaはその誕生から今日に至るまで、常に進化を続けてきました。  
初期のWebアプレットからエンタープライズシステム、そして現代のクラウドネイティブなアプリケーション開発まで幅広い分野でその地位を確立しています。  

半年ごとのリリースサイクルとLTS版の提供により、最新の技術トレンドを取り入れつつも、長期安定稼働を求めるシステムにも対応できる柔軟なエコシステムを維持しています。  
Javaは現代の要求に応える多くの機能を取り込み、開発者にとって強力なツールであり続けています。  
今後もJavaの進化は続き、ソフトウェア開発を支え続けるでしょう。
