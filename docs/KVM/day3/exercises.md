## Part 2：障害調査ロールプレイ

実際に `virsh` コマンドを使って VM の状態を調査します。  
各シナリオの手順に従いコマンドを実行し、調査結果を記録してください。

> **前提：** Day 2 で作成した `almalinux01` が存在すること。  
> 存在しない場合は Day 2 の手順を参考に再作成してください。

---

### シナリオ 1：VM の状態を調査する

**状況：** 上司から「almalinux01 の状態を確認して報告してほしい」と言われました。

以下のコマンドを順番に実行し、出力を確認してください。

**Step 1：VM の基本情報を取得する**

```bash
sudo virsh dominfo almalinux01
```

以下の項目を出力から読み取り、記録してください。

| 項目 | 値 |
|------|-----|
| 現在の状態（State） | |
| 割り当て vCPU 数 | |
| 最大メモリ | |
| 使用メモリ | |

- [ ] 情報を読み取れた

**Step 2：vCPU の割り当て状況を確認する**

```bash
sudo virsh vcpuinfo almalinux01
```

- [ ] どの物理 CPU に vCPU がスケジュールされているか確認できた

**Step 3：ネットワーク設定を確認する**

```bash
sudo virsh domiflist almalinux01
```

- [ ] 接続しているネットワーク名とデバイス種別（virtio など）を確認できた

**Step 4：XML 定義を確認する**

```bash
sudo virsh dumpxml almalinux01 | grep -E '<memory|<vcpu|<disk'
```

- [ ] メモリ・vCPU・ディスクの定義行が表示された

---

### シナリオ 2：ネットワークが起動していない

**状況：** VM を起動しようとしたが、「network 'default' is not active」というエラーが出ました。

**Step 1：ネットワーク一覧を確認する**

```bash
sudo virsh net-list --all
```

`default` の State を確認してください。

| ネットワーク名 | State | Autostart |
|--------------|-------|-----------|
| default | | |

- [ ] 確認できた

**Step 2：ネットワークを起動する**

State が `inactive` の場合、以下で起動してください。

```bash
sudo virsh net-start default
```

**Step 3：自動起動を有効にする**

```bash
sudo virsh net-autostart default
```

**Step 4：再度確認する**

```bash
sudo virsh net-list --all
```

- [ ] `default` が `active` かつ `Autostart: yes` になった

---

### シナリオ 3：VM のリソースを変更する

**状況：** `almalinux01` のメモリが不足気味です。現在の設定を確認したうえで、メモリ上限を変更してください。

> WSL2 環境では `--live` オプション（稼働中の変更）が使えない場合があります。  
> 停止状態で `--config` による設定変更を実施してください。

**Step 1：現在のメモリ設定を確認する**

```bash
sudo virsh dominfo almalinux01 | grep -i memory
```

現在の最大メモリ：＿＿＿＿ KiB（＿＿＿＿ MB）

- [ ] 確認できた

**Step 2：VM を停止する（起動中の場合）**

```bash
sudo virsh shutdown almalinux01
sudo virsh list --all   # shut off になるまで待つ
```

**Step 3：最大メモリを 3 GB に変更する**

```bash
sudo virsh setmaxmem almalinux01 3G --config
sudo virsh setmem almalinux01 3G --config
```

**Step 4：設定が反映されたか確認する**

```bash
sudo virsh dominfo almalinux01 | grep -i memory
```

- [ ] Max memory が `3145728 KiB`（= 3 GB）になっていることを確認できた
