
# プロンプトエンジニアリング

AIは「質問の意味」を理解しているのではなく、  「次に来る言葉を確率で予測しているだけ」です。

だからこそ、AI任せにせず、 
人間が **書き出し・条件・前提** を与え、  
予測の方向性をガイドする必要があります。

![AI](../img/prompt.jpg)

---

## 2.1 伝わらない指示 vs 伝わる指示

プロンプトが曖昧だと、  
AIは「無難で平均的な回答」しか出せません。

### 指示の違いによる結果の差

| 指示内容 | AIの内部状態 | 出力結果 |
|---|---|---|
| メールを書いて | 誰に？何の目的？→分からない | 使えない一般文 |
| 取引先への謝罪メール。理由はシステム障害。誠実かつ簡潔に | 文脈が明確 | 実務で使える下書き |

重要なのは「細かく書くこと」ではなく、  **人間が考えた前提をAIに渡すこと**です。

![AI](../img/good-bad.jpg)

## 2.2 プロンプトの型（4要素）

毎回ゼロから考える必要はありません。  
以下の **4つの要素** を入れるだけで、出力の品質を安定させることができます。

### プロンプトの基本構成（4要素）

| 要素 | 役割 |
|---|---|
| 役割（Role） | AIにどの立場で考えさせるか |
| 目的（Purpose） | 何のための文章か |
| 条件（Condition） | 制約・ルール |
| 形式（Format） | 出力の見た目・使い道 |


:::info
### 実践テンプレート
以下のプロンプトをChatGPTやGeminiなどの汎用AIツールに<br />コピー＆ペーストして生成結果を確認してましょう。
```text
指示文
あなたは [役割：ベテランの営業部長] です。
部下に向けて [目的：目標達成を激励するメッセージ] を作成してください。

条件
[文字数：200文字程度]
[トーン：熱意を持って、かつ親しみやすく]
[含める内容：前四半期の成果への感謝]
出力形式
チャットツールで送信できるテキスト形式
```
:::

## 2.3 回答の質を上げる「魔法の一言」

一度で完璧な回答を求める必要はありません。  
AIは **対話しながら調整する** ことで真価を発揮します。

![AI](../img/magic.jpg)

その際、以下のようなフレーズをプロンプトに追加すると、より的確で質の高い回答が得られます。
### よく使う追加指示


| 追加指示 | 目的・効果 | 活用イメージ |
|---|---|---|
| ステップ・バイ・ステップで考えて | 手順を分解させ、論理ミスや抜け漏れを減らす | 業務フロー整理、設計手順の説明 |
| 初心者にもわかるように説明して | 専門用語を噛み砕き、平易な表現へ変換する | 研修資料、マニュアル作成 |
| 要点を3つに絞って | 情報量を整理し、重要ポイントだけを抽出する | 上司向けサマリ、スライド原稿 |
| 表形式にまとめて | 情報を比較しやすく整理し、視認性を高める | 仕様比較、整理メモ |
| 逆の視点（デメリット）から批判して | リスクや反論を洗い出し、思考の偏りを防ぐ | 企画レビュー、意思決定前の検討 |
| 具体的な事例を挙げて | 抽象的な内容を具体化し、理解と説得力を高める | 研修説明、提案資料 |

## 2.4 プロンプト設計の考え方

生成AIに慣れてくると、プロンプトは文章形式で自由に書けるようになります。
しかし、使い始めたばかりの段階では「何を書けばよいのか」で迷いやすいポイントでもあります。

そのため、最初は感覚で書くのではなく、
**役割・目的・条件・形式**の4つに分解して考える方法をおすすめします。
これは、AIが迷わないように情報を整理し、確率の幅を狭めるための基本的な考え方です。

**プロンプト設計を行う際は、以下のフォーマットを参考にご使用ください。**
:::info

## 設計シート（記入用フォーマット）

```text
【役割】
- どの立場・視点で書くか

【目的】
- いつの内容か：
- 誰に対して：
- 何を伝えるためか：
- 相手にどうしてほしいか：

【条件】
- 文字数・分量：
- トーン（雰囲気）：
- 構成の指定：
- 必ず含める要素：
- 避けたい表現・NG事項：

【形式】
- 提出・利用シーン：（メール／Word／PDF／チャット など）
- 構造：（件名＋本文／見出し付き／箇条書き中心 など）
```
:::


※演習課題ページにも同じフォーマットを載せているため、保存する必要はありません。

このように項目を整理してからプロンプトを作成することで、  
**要件漏れやトーンのずれを防ぎやすくなります。**

また、ケース別にこれらのプロンプトを事前に準備しておくことで、  
業務をより円滑に進めることができます。

ここからは、さまざまな業務シーンを想定した演習を通して、それぞれのプロンプト設計を学んでいきます。
