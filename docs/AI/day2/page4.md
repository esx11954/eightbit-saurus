# 異常値・改善ポイントの発見

この章では、大量のデータをただ確認するだけで終わらせず、<br />
その中に含まれる**通常とは異なる数値の動きや傾向**を捉え、<br />
原因についての仮説を整理し、次の判断につなげる視点を身につけます。

![AI](./img/point.jpg)

## 演習9-1　分析前処理のためのデータクレンジング

| 項目 | 内容 |
|---|---|
| ケース | 演習8-6で使用した売上データを確認したところ、店舗名の表記揺れ（全角・半角、略称混在）や、未入力のセルが含まれていることが分かりました。上司から「このまま分析に使うのは不安だから、まずデータをきれいにしてほしい」と依頼されました。 |
| 演習内容 | 上司の依頼をCopilot向けに翻訳し、データ分析に使用できる状態に整形してください。<br />どの列をどのようなルールで整形すべきかを明確にしたうえで、表記揺れや空白セルを整理させることを目的とします。具体的な関数や操作手順は決め打ちせず、やりたい業務内容からAIに処理方針を考えさせてください。 |
| 提出物 | ① 指示文<br />② クレンジング後のデータを含むExcelファイル |
| 留意点 | ・元データの数値を勝手に変更していないか<br />・どのようなルールで整形したのかが指示文から読み取れるか |

## 演習9-2　異常値の発見

| 項目 | 内容 |
|---|---|
| ケース | 演習9-1でクレンジングしたデータを用いて月次売上を確認したところ、特定の店舗だけ、ある月から売上が大きく落ち込んでいることに気づきました。上司から「どこで異常が起きているのか整理してほしい」と依頼されました。 |
| 演習内容 | 上司の依頼をCopilot向けに翻訳し、通常と異なる傾向を示しているデータを特定してください。<br />異常と考えられる店舗や月を、数値やグラフを根拠に指摘させることを目的とします。原因の断定までは行わず、「どこで何が起きているか」を明確にしてください。 |
| 提出物 | ① 指示文<br />② 異常値が分かる集計結果またはグラフ |
| 留意点 | ・「異常」と判断した根拠が明示されているか<br />・断定的な表現になっていないか |

## 演習9-3　異常値に対する原因仮説の整理

| 項目 | 内容 |
|---|---|
| ケース | 演習9-2で発見した売上の落ち込みについて、上司から「なぜこうなっているのか、考えられる理由を整理してほしい」と依頼されました。ただし、原因はまだ確定していません。 |
| 演習内容 | 上司の依頼をCopilot向けに翻訳し、異常値に対して考えられる原因を仮説として整理させてください。<br />季節性、外部要因、業務上の変化など、複数の観点から可能性を洗い出させることを目的とします。結論を断定せず、「仮説」として整理させてください。 |
| 提出物 | ① 指示文<br />② AIによる仮説整理の出力結果 |
| 留意点 | ・原因を断定していないか<br />・データやグラフを根拠とした仮説になっているか |

**この章でできるようになること**
- 人の目では見逃しやすい異常に気づける  
- 数字の変化を「原因を考える材料」として扱える
