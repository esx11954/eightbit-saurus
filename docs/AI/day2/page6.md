# 追加演習

## 演習8-6：表記ゆれの統一

| 項目 | 内容 |
|---|---|
| ケース | 複数の支店から顧客リストを集めましたが、入力ルールが守られておらずバラバラです。「（株）」と「株式会社」、「１丁目」と「1丁目」、氏名のスペース有無などが混在しており、このままではDM送付システムに取り込めません。 |
| 演習内容 | 上司の依頼をCopilot向けに翻訳し、以下の4つの要件をすべて満たすようにデータを整形してください。<br />・ 社名の「（株）」や「(株)」はすべて「株式会社」に統一されていること。<br />・ 住所の中にある数字（全角）はすべて「半角」に置換されていること。<br />・氏名の苗字と名前の間にスペースがない場合、全角スペースが挿入されていること。<br />・ 住所から「都道府県」だけが抜き出され、別列（B列）に配置されていること。 |
| 提出物 | ① 4つの要件を矛盾なく整理し、Copilotに指示するためのプロンプト<br />② 要件を満たした整形後の顧客リスト |
| 留意点 | 「きれいにして」といった抽象的な指示ではAIは動きません。どの文字をどう扱うのか、処理要件を具体的に言語化できているかを確認してください。 |

## 演習8-7：ノイズ除去と実質パフォーマンス分析

| 項目 | 内容 |
|---|---|
| ケース | 担当者別の売上集計を行いましたが、単純な合計ではAさんの成績が不当に低く見えます。データを確認すると、Aさんは「返品処理（マイナス売上）」や「テスト入力（0円〜10円）」のデータが大量に含まれていました。上司から「そういうノイズを除外した、実質の平均客単価を出して」と依頼されました。 |
| 演習内容 | 単純なAVERAGEではなく、次の考え方を満たす形で集計結果を出すようにCopilotへ指示してください。<br />・ 売上金額が「0以下」または「100円未満」のデータは、分析上のノイズとして扱われていること。<br />・ ノイズを除外したうえで、担当者ごとの「平均客単価」が算出されていること。<br />・ 併せて、集計対象となった「有効取引件数」も確認できること。 |
| 提出物 | ① ノイズの定義と集計方針が明確に書かれた指示文<br />② ノイズ除去後の集計結果表 |
| 留意点 | データ分析では「計算」よりも前に、何を分析対象から外すかの判断が重要です。除外条件が曖昧になっていないかを確認してください。 |

## 演習8-8：請求と入金の照合

| 項目 | 内容 |
|---|---|
| ケース | 経理事務の定番業務です。「請求一覧（誰にいくら請求したか）」と「銀行入金明細（誰からいくら振り込まれたか）」の2つのシートがあります。上司から「まだ入金されていない請求（未回収）がどれかリストアップして」と依頼されました。 |
| 演習内容 | 異なる2つのシートを突き合わせ、次の状態が確認できるようにCopilotへ指示してください。<br />・ 請求一覧の各行について、入金が確認できたかどうかが判別できること。<br />・ 金額が一致する場合は「消込済」と判断されていること。<br />・ 対応する入金が見つからない場合は「未入金」と判定されていること。<br />・ 顧客名の表記ゆれによる誤判定リスクについて、Copilotが注意を促していること。 |
| 提出物 | ① 2つの表を照合する際の考え方（キーの考え方）を含めた指示文<br />② 消込結果と未入金請求が分かる一覧 |
| 留意点 | 「何を基準に一致とみなすか」「一致しなかった場合どう扱うか」という突合ロジックが明確かを確認してください。 |

## 演習8-9：複数条件によるステータス自動判定

| 項目 | 内容 |
|---|---|
| ケース | 営業案件リスト（数千件）があり、各案件には「見積日」「現在のステータス（見積済/受注/失注）」「最終連絡日」が記録されています。上司から「放置されている案件だけを抽出して、至急フォローさせたい」と依頼されました。 |
| 演習内容 | 以下の条件を満たす案件が判別できるように、Copilotへ指示してください。<br />・ ステータスが「見積済」のままであること。<br />・ 最終連絡日から14日以上経過していること。<br />・ ただし、備考欄に「来月検討」と記載されている案件は対象外となっていること。<br />・ 条件を満たす案件には「要フォロー」と分かる表示がされていること。 |
| 提出物 | ① AND条件・除外条件が整理された指示文<br />② フラグ付けと強調表示が反映された案件リスト |
| 留意点 | 条件が複数ある場合、必須条件と除外条件が混同されていないかを確認してください。例外条件の扱いが精度を左右します。 |
